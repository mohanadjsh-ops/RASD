# Internationalization

Rasd supports Arabic and English from the first version.

- Arabic route prefix: `/ar`
- English route prefix: `/en`
- Arabic layout direction: `rtl`
- English layout direction: `ltr`
- Arabic strings: `messages/ar.json`
- English strings: `messages/en.json`

UI components should receive translated labels or load messages server-side. Do not hardcode display strings directly inside reusable components when a translated value is available.

Dashboard content can preserve source article titles in their original language. Newsroom output is Arabic by default, with architecture prepared for English output later.
